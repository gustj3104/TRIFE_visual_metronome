첨부한 Android Music Player with Spinning Gramophone 레퍼런스의 전체적인 UI 구성과 인터랙션 감성을 참고하여, 청각장애인과 비청각장애인이 함께 박자를 맞출 수 있는 웹 기반 ‘비주얼 메트로놈’ 서비스의 메인 화면을 디자인해줘. 레퍼런스에서 다음 요소를 적극적으로 참고한다. * 화면 중앙에 하나의 핵심 비주얼 오브젝트가 크게 배치되는 구조 * 사용자가 처음 봤을 때 바로 인지할 수 있는 크고 명확한 원형 시작 버튼 * 버튼을 눌렀을 때 핵심 오브젝트가 즉시 움직이기 시작하는 반응형 애니메이션 * 재생과 정지 상태가 시각적으로 명확하게 구분되는 인터랙션 * 넓은 여백과 적은 수의 UI 요소를 사용하는 미니멀한 디자인 * 핵심 기능에 시선을 집중시키는 단순한 화면 위계 * 둥근 버튼, 부드러운 상태 전환, 절제된 그림자와 배경 깊이감 * 사용자가 별도의 설명 없이도 시작 버튼과 핵심 비주얼의 관계를 이해할 수 있는 구조 단, 레퍼런스의 축음기, 음반, 음악 플레이어 기능이나 앨범 커버는 그대로 사용하지 않는다. 레퍼런스의 화면 구성, 시작 버튼, 인터랙션 방식, 시각적 집중 구조만 참고하여 완전히 새로운 비주얼 메트로놈 서비스로 설계한다. ## 서비스 목적 이 서비스는 청각장애인과 비청각장애인이 춤 연습, 움직임 워크숍, 공연 준비 과정에서 하나의 프로젝터 화면을 함께 바라보며 동일한 박자와 카운트를 공유할 수 있도록 하는 접근성 중심의 시각적 리듬 도구다. 음악을 듣기 어려운 사용자도 화면 중앙의 반복적인 움직임과 도형 크기 변화를 통해 다음 박자의 도착 시점, 현재 박자, 첫 박자를 직관적으로 인지할 수 있어야 한다. 주요 사용 환경은 다음과 같다. * 16:9 프로젝터 화면 * 댄스 연습실 * 움직임 워크숍 * 공연장 * 여러 사람이 3m 이상 떨어져 하나의 화면을 함께 보는 상황 ## 핵심 화면 레이아웃 데스크톱과 16:9 프로젝터 환경을 기준으로 화면을 크게 두 영역으로 나눈다. ### 왼쪽 메인 비주얼 영역 * 전체 화면 너비의 약 72~78% * 화면 중앙에 핵심 비주얼 메트로놈을 크게 배치 * 단순한 원, 공, 바, 기준선 등으로 구성 * 다른 장식이나 텍스트가 핵심 비주얼을 방해하지 않도록 함 * 사용자가 가장 먼저 시각적 리듬 오브젝트를 인지해야 함 * 재생 중에는 핵심 비주얼이 일정한 BPM에 맞춰 움직임 * 정지 중에는 비주얼이 안정적인 대기 상태를 유지 * BPM과 현재 카운트는 비주얼 주변에 보조 정보로 배치하되 과도하게 크지 않게 함 * 컨트롤 패널이 접히면 메인 비주얼 영역이 화면 전체로 확장됨 ### 오른쪽 컨트롤 패널 * 전체 화면 너비의 약 22~28% * 화면 우측 전체 높이를 사용하는 고정형 세로 패널 * 메인 비주얼과 분리되지만 동일한 디자인 시스템을 사용 * 불투명한 큰 관리자 대시보드처럼 보이지 않도록 함 * 미니멀한 패널 안에 주요 조작 기능을 세로로 정돈 * 내부 기능을 기능별 그룹으로 구분하되 카드가 너무 많아 보이지 않도록 함 * 사용자가 위에서 아래로 자연스럽게 조작할 수 있는 순서를 사용 * 패널은 접고 펼칠 수 있어야 함 * 접힌 상태에서는 화면 우측에 작은 세로 핸들 또는 화살표만 남김 ## 초기 대기 화면 서비스에 진입하면 화면 중앙에 비주얼 메트로놈의 대기 상태와 큰 시작 버튼을 보여준다. ### 시작 버튼 첨부 레퍼런스의 메인 재생 버튼처럼 다음 특징을 적용한다. * 화면 중앙 비주얼 바로 아래 또는 비주얼 중심에 배치 * 크고 명확한 원형 버튼 * 재생 아이콘과 Start 텍스트를 함께 사용하거나, 재생 아이콘을 중심으로 구성 * 멀리서도 시작 버튼임을 즉시 인지할 수 있어야 함 * 마우스 Hover 시 버튼이 약 3~5% 확대 * Hover 시 그림자와 테두리가 부드럽게 변화 * Pressed 상태에서는 버튼이 살짝 축소되어 눌리는 감각을 전달 * 클릭 시 재생 아이콘이 일시정지 아이콘으로 자연스럽게 전환 * 버튼 클릭과 동시에 중앙 비주얼의 움직임이 시작 * 키보드 Space 입력으로도 동일하게 시작 또는 정지 * 버튼 상태 변화는 150~250ms 정도의 짧고 명확한 애니메이션으로 표현 버튼 애니메이션은 장식적인 효과가 아니라 사용자의 입력이 정상적으로 처리됐음을 확실하게 알려주는 피드백 역할을 해야 한다. ## 중앙 핵심 비주얼 메인 비주얼은 레퍼런스의 회전하는 음반처럼 서비스의 상태와 움직임을 가장 직접적으로 보여주는 핵심 오브젝트다. 기본 시각화는 Bounce 타입으로 설계한다. ### Bounce 타입 * 큰 공 하나가 화면 중앙의 수직 축을 따라 위아래로 움직임 * 또는 가로 축을 따라 좌우로 움직이는 방향 선택 가능 * 움직임은 중력 기반이 아닌 일정한 선형 속도 * 각 이동 구간에서 속도가 일정해야 함 * 공이 기준선 또는 끝 지점에 도착하는 순간을 한 박자로 정의 * 도착 순간 공의 크기가 짧게 확대됨 * 일반 박자에서는 약한 크기 변화 * 첫 박자에서는 일반 박자보다 확실히 큰 크기 변화 * 색상만으로 박자를 구분하지 않음 * 크기, 위치, 기준선 두께 변화로 박자를 표현 * 움직임의 궤적을 따라 과도한 잔상이나 빛 효과를 사용하지 않음 공과 기준선은 멀리서도 명확하게 식별할 수 있을 만큼 크고 단순해야 한다. ### 대기 상태 메트로놈이 시작되기 전에는 다음과 같이 표현한다. * 공은 화면 중앙 또는 시작 지점에 정지 * 기준선은 고정 * 현재 BPM 숫자는 표시 가능 * 비주얼 전체가 아주 약하게 호흡하듯 커졌다 작아지는 대기 애니메이션을 적용할 수 있음 * 대기 애니메이션은 실제 박자로 오인되지 않을 만큼 느리고 약하게 표현 * 중앙 시작 버튼이 가장 명확한 조작 요소로 보여야 함 ### 재생 상태 시작 버튼을 누르면 다음 변화가 동시에 일어난다. * 공이 선형 움직임 시작 * 버튼 아이콘이 Pause로 전환 * 우측 패널의 상태 텍스트가 Playing으로 변경 * 현재 카운트가 1부터 순차적으로 진행 * 첫 박자에서 공의 크기가 더 크게 변화 * BPM 조절 시 애니메이션 속도가 즉시 변경 * 화면 전환 없이 모든 변화가 같은 화면에서 일어남 ### 일시정지 상태 * 공이 현재 위치에서 멈춤 * 시작 버튼이 다시 Play 아이콘으로 변경 * 우측 패널 상태가 Paused로 변경 * 현재 카운트 상태는 유지 * 다시 재생하면 현재 상태에서 이어가거나 첫 박자부터 다시 시작하는 옵션을 제공할 수 있음 * 기본 동작은 첫 박자부터 다시 시작하는 것으로 설계 ## 카운트 표시 중앙 비주얼 근처에 현재 카운트를 크게 표시한다. ### 4/4 모드 * 1, 2, 3, 4 반복 * 현재 숫자를 가장 크게 표현 * 나머지 숫자는 작거나 낮은 명도로 표시 * 첫 박자인 1에서 공의 크기 변화가 가장 크게 나타남 예시: 1 2 3 4 ### 8카운트 모드 * 1부터 8까지 반복 * 현재 숫자를 크게 표현 * 첫 번째 카운트를 가장 강하게 강조 * 5번째 카운트에는 약한 보조 구분만 적용 가능 * 1번보다 강하게 표현하지 않음 예시: 1 2 3 4 5 6 7 8 카운트 숫자는 중앙 비주얼의 움직임을 방해하지 않도록 비주얼 상단, 하단 또는 원형 궤도 주변에 배치한다. ## 우측 컨트롤 패널 상세 구성 컨트롤 패널은 다음 순서로 구성한다. ### 1. 상단 상태 영역 * 서비스명 또는 간결한 로고 * 현재 상태: Ready, Playing, Paused, Analyzing * 컨트롤 패널 접기 버튼 * 라이트 또는 다크 모드 전환 버튼 * 전체 화면 버튼 서비스명과 상태 표시가 너무 많은 공간을 차지하지 않도록 한다. ### 2. BPM 컨트롤 BPM을 패널에서 가장 중요한 정보로 표현한다. * 큰 BPM 숫자 * 작은 BPM 라벨 * − 버튼 * * 버튼 * 숫자 직접 입력 * 세로 또는 가로 슬라이더 * Tap Tempo * 현재 범위 40–240 BPM 예시: 120 BPM BPM 숫자를 드래그하거나 마우스 휠로 스크롤하여 값을 조절하는 인터랙션도 고려한다. * 위로 드래그 또는 스크롤: BPM 증가 * 아래로 드래그 또는 스크롤: BPM 감소 * 값 변경 시 숫자가 부드럽게 교체 * BPM 변경과 동시에 중앙 비주얼의 속도가 즉시 반응 ### 3. 카운트 모드 * 4/4 * 8 Count 두 옵션을 Segmented Control 형태로 구성한다. 선택된 모드는 배경, 테두리, 굵기, 체크 상태 등으로 명확하게 구분한다. 색상만으로 선택 여부를 전달하지 않는다. ### 4. 시각화 타입 다음 네 가지 타입을 선택할 수 있게 한다. * Bounce * Swing * Pulse * Sweep 각 타입은 작은 미리보기 아이콘과 텍스트 라벨을 함께 사용한다. 선택 즉시 중앙 비주얼이 해당 타입으로 전환된다. 전환 시 비주얼이 갑자기 사라졌다 나타나기보다 짧은 페이드 또는 형태 변환 애니메이션을 적용한다. ### 5. 움직임 설정 * Vertical * Horizontal * Motion intensity: Low, Standard, High * First beat emphasis On / Off * Count number On / Off 기본 설정은 다음과 같다. * Vertical * Standard * First beat emphasis On * Count number On ### 6. 컬러 프리셋 자유 컬러피커는 제공하지 않는다. 접근성과 프로젝터 가시성이 검증된 프리셋만 제공한다. * White / Black * Black / White * Black / Yellow * Navy / White * White / Deep Blue * Dark Purple / Lime * Burgundy / Cream 각 프리셋은 작은 원형 또는 사각형 색상 샘플로 표현한다. 선택된 프리셋에는 체크 표시와 굵은 테두리를 함께 적용한다. ### 7. 시작 카운트 * Immediately * 4 Count * 8 Count * 5, 6, 7, 8 드롭다운보다 버튼 그룹이나 Segmented Control을 우선 사용한다. ### 8. BPM 자동 감지 컨트롤 패널 하단에 다음 기능을 배치한다. * Upload audio file * Analyze device audio * Analyze again * Detected BPM * Confidence * Apply BPM 자동 감지 기능은 메인 BPM 설정보다 시각적 우선순위를 낮게 둔다. ## BPM 자동 감지 진입 화면 최초 진입 시 BPM 설정 방법을 선택하는 간결한 오버레이 또는 중앙 선택 화면을 제공할 수 있다. 선택지는 다음 세 가지다. * Upload audio file * Analyze device audio * Set BPM manually 선택 버튼들은 레퍼런스의 시작 버튼처럼 크고 명확하되, 세 개가 동일하게 강한 원형 버튼으로 보이지 않도록 한다. 권장 구조: * Set BPM manually: 가장 중요한 Primary 버튼 * Upload audio file: Secondary 버튼 * Analyze device audio: Secondary 버튼 음원 분석을 선택한 경우 분석 결과를 적용하기 전까지 우측 컨트롤 패널의 주요 기능을 비활성화한다. 결과 적용 후 전체 컨트롤이 활성화된다. ## 버튼 인터랙션 첨부 레퍼런스처럼 모든 버튼이 눌렸을 때 즉각적인 시각 반응을 제공한다. ### Primary 버튼 * Hover: 3~5% 확대 * Hover: 그림자와 명도 증가 * Pressed: 약 2~4% 축소 * Release: 부드럽게 원래 크기로 복귀 * Selected: 배경 또는 테두리 상태 유지 * Disabled: 명도 감소, 움직임 제거 * Loading: 아이콘이 로딩 상태로 전환 ### 아이콘 버튼 * Hover 시 배경에 작은 원형 또는 둥근 사각형 표면 생성 * Pressed 시 축소 * 선택 상태에서는 배경과 테두리 유지 * 모든 아이콘 버튼에 Tooltip 제공 ### 시작 또는 재생 버튼 * 가장 강한 시각적 위계 * 원형 * Play와 Pause 상태 전환 * 클릭과 중앙 비주얼 애니메이션이 정확히 동시에 반응 * 버튼 자체가 계속 회전하거나 과도하게 움직이지 않음 * 실제 재생 상태가 중앙 비주얼을 통해 드러나도록 함 ## 전체 화면 프로젝터 모드 전체 화면을 실행하면 다음 요소만 남긴다. * 중앙 핵심 비주얼 * 현재 카운트 * 필요한 경우 작은 BPM 표시 * 우측 컨트롤 패널 핸들 컨트롤 패널과 기타 버튼은 일정 시간 후 숨겨진다. 마우스를 움직이거나 화면을 클릭하면 다시 나타난다. 프로젝터 모드에서는 핵심 비주얼이 화면의 약 60~75% 높이를 사용하도록 한다. ##
